Olá meu nome é Hitalo Filipe e neste breve vídeo vou apresentar a prova de conceito do 
sistema SIGO  (Sistema Integrado de Gestão e Operação) e alguns outros aspectos relacioados
ao processo de planejamento e desenvolvimento do projeto.


O codigo fonte está disponivel no GitHub, os módulos das API's, a aplicação frontend,
o API gateway, os bancos de dados, o servidor RabbitMQ e as imagens dos containers estão 
hospedados no Azure. Com excessão dos bancos de dados, todos os servidores estão em containers.
E cada um deles esta em uma imagem docker exclusiva.

os pipelines de integração continua e de entrega continua estão no Azure devops.

pq nao usar apenas o github ou apenas o azure? pq pretendo deixar o projeto no meu portifolio
publico no GitHub e ja tenho experiencia com azure. Alem de que acho importante usar multiplos
serviços de nuvem. Uma possível evolução do processo de implantação poderia ser o uso de mais
de um serviço, como por exemplo o AWS.

O ESB não foi implantado, ele está sendo simulado na infrestrutura local com uma aplicação .NET Core.
